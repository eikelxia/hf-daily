# 开店助手 (hf-daily) — 技术方案文档

## 一、系统概述

开店助手是一个基于 GitHub Actions 的自动化门店开店项目管理工具。输入开业日期和门店面积，自动生成 37 个节点的时间线，每天定时通过企业微信群机器人发送提醒，数据自动同步到企业微信智能表格。

## 三、核心业务流程

### 3.1 创建项目

```
[用户] 打开自建应用 → 点击「新建项目」
         ↓
   填写：项目名称、门店名称、开业日期、门店面积
         ↓
   页面本地计算 37 个节点的完整时间线（预览）
         ↓
   点击「复制 JSON」→ 点击「打开 GitHub 编辑」
         ↓
   在 GitHub 页面粘贴 → Commit
         ↓
   GitHub Actions 自动触发扫描
         ↓
   群消息通知 + 智能表格同步
```

### 3.2 每日定时提醒

```
GitHub Actions 每天 9:00 和 14:00 自动触发
         ↓
   读取 data/projects.json
         ↓
   遍历所有活跃项目的所有节点
         ↓
   识别需要提醒的节点：
   ├─ start_confirm：明天应开始，提前1天提醒
   ├─ completion_check：今天应完成，当天验收
   └─ daily_chase：已超期，每天追问
         ↓
   通过企业微信群机器人发送 Markdown 消息
         ↓
   同步最新状态到智能表格
```

### 3.3 确认操作

```
[群消息] 点击「确认开始」或「确认完成」链接
         ↓
   阿里云 FC HTTP 函数接收请求
         ↓
   调 GitHub API 读取 data/projects.json
         ↓
   更新对应节点状态
         ↓
   写回 GitHub
         ↓
   返回确认成功页面
```

### 3.4 数据查看

```
[任何人] 打开企业微信智能表格
    → 查看各项目各节点状态（权限由企微控制）

[管理员] 打开自建应用
    → 进入 GitHub Pages 看板，查看全局视图
```

## 六、维护指南

### 创建新项目
1. 打开自建应用 → 新建项目 → 填表 → 生成时间线
2. 复制 JSON → 打开 GitHub 编辑链接 → 粘贴保存

### 确认节点状态
- 在群消息中点击「确认开始」或「确认完成」链接即可

### 修改节点模板
- 编辑 `scripts/daily_scan.py` 中的 `NODE_TEMPLATES`
- 编辑 `docs/create.html` 中的 `templates`（前后端要保持一致）
- Push 到 GitHub 后自动生效

### 修改提醒时间
- 编辑 `.github/workflows/daily-scan.yml` 中的 `cron` 表达式
- 目前：北京时间每天 9:00 和 14:00
